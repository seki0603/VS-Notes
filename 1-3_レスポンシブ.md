# レスポンシブデザイン

画面サイズが異なるデバイス毎に最適な表示に自動的に切り替えるための制作手法

<br>

## メディアクエリ

CSS の記述に条件を加える記法  
画面サイズの変化に応じて、適用される CSS を切り替えることができる。  
正しく動作させるために HTML に viewport の指定が必要  
`<meta name="viewport" content="width=device-width, initial-scale=1.0">`

<br>

## ブレイクポイント

デザインが切り替わる画面幅のポイント  
画面幅のシェア状況から、下記ブレイクポイント推奨。

- スマートフォン：~480px
- タブレット：~768px
- PC：769px~

<br>

## レスポンシブデザイン手法

### デスクトップファースト

デスクトップ画面を基準に作り、スマホ・タブレット画面をメディアクエリ指定する方法。  
メディアクエリにはスマホ画面とタブレット画面のそれぞれ特有の CSS だけ記述する。

```
p {
  color: black;
}

@media screen and (max-width: 768px) {
  p {
    color: blue;
  }
}

@media screen and (max-width: 480px) {
  p {
    color: red;
  }
}
```

max-width = 最大幅を指定するプロパティ

### モバイルファースト

スマホ画面を基準に作り、タブレット・デスクトップ画面をメディアクエリで指定する記法。  
タブレット画面とデスクトップ画面のそれぞれ特有の画面だけ記述する。

```
p {
  color: black;
}

@media screen and (min-width: 481px) {
  p {
    color: red;
  }
}

@media screen and (min-width: 769px) {
  p {
    color: blue;
  }
}
```

# レスポンシブデザインパターン

## 要素を拡大縮小させるパターン

画面幅の大きさに合わせて、要素の大きさを拡大縮小させるパターン。

- 縦横比を維持して拡大縮小
- 縦横比を維持せず拡大縮小
- メディアクエリで拡大縮小

<br>

### 縦横比を維持して拡大縮小

擬似要素である before を使用して、padding の%が親要素を基準としている  
という特性を利用する。  
<br>
縦横比を３：４に維持して拡大縮小する例

```
.item{
　width: 100%;
  position:relative;
}
.item::before{
  content:"";
  display: block;
  padding-top: 75%;
}

.item-content{
  position: absolute;
  top: 0;
  left: 0;
  bottom: 0;
  right: 0;
  background-color: yellow;
}
```

<br>

### 縦横比を維持せず拡大縮小

コンテンツの縦幅を基準にして拡大縮小する  
画像などで使用されることが多い  
<br>

`<img class="item" src="img/red.jpg" alt="">`

```
.item {
  width: 100%;
  height: auto;
}
```

<br>

### メディアクエリで拡大縮小

メディアクエリによって、デバイス毎の要素の大きさを変える。
<br>

## 別の要素を表示するパターン

スマホ版とパソコン版の要素を２つ用意しておき、それぞれの時に  
要素の表示と非表示を切り替えるパターン。  
要素を２つコーディングしておいて、メディアクエリで`display: none;`  
によって、表示と非表示を切り替える。

```
<p class="item1">要素1</p>
<p class="item2">要素2</p>
```

```
.item2 {
  display: none;
}

@media screen and (max-width: 768px){
  .item1{
    display: none;
  }
  .item2{
    display: block;
  }
}
```

<br>

## レイアウトで折り返すパターン

画面幅が大きいときは横に並べておき、  
幅が小さいときは縦に並べるパターン。  
Flexbox で実装

```
<div class="parent">
    <div class="child"></div>
    <div class="child"></div>
    <div class="child"></div>
</div>
```

```
.parent {
  display: flex;
  justify-content: space-between;
  flex-wrap: wrap;
}
.child {
  width: 33%;
  height: 200px;
  background: #ff0000;
}

@media screen and (max-width: 768px){
  .child {
    width: 49%;
  }
}

@media screen and (max-width: 480px){
  .child {
    width: 100%;
  }
}
```

# レイアウト

## シングルカラムレイアウト

サイドにコンテンツを設けず、１つのカラムを縦に並べていくレイアウト。

<br>

## マルチカラムレイアウト

ページを複数の列に分割して配置するレイアウト

- ２カラム　ページをメニューとメインコンテンツに分割する
- ３カラム　ナビ、メイン、補助情報など

## グリッドレイアウト

それぞれの要素をグリッド（格子）のように並べることで、  
美しく整えられた印象のデザインを作るレイアウト。  
EC サイトや SNS 向け

`column-count: 値;`　列数（縦割りの数）を親要素に指定  
`break-inside: avoid;`　要素が途中で分割されないように子要素に指定

<br>

## フルスクリーンレイアウト

イメージ写真や動画をブラウザの幅いっぱいに映し出したデザインのレイアウト  
写真のギャラリーサイトやブランドイメージを前面に出したい各種プロダクト向け

```
.mv{
    position: relative;
    background: url(画像へのパス) no-repeat center/cover;
    height: 100vh;
    width: 100vw;
}
.title{
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    font-size: 64px;
    color: #fff;
}
```

# Figma

1. 新規ページ作成  
   ドラフト＋　 → 　デザイン  
   <br>

2. 画像の読み込み  
   メニュー　 → 　ファイル　 → 　画像の配置  
   <br>

3. コンテンツの切り分け  
   下部のシェイプツールから長方形を選択　 → 　ドラックでコンテンツを囲む  
   塗りを０％にして線を追加
   <br>

4. ブロックの中にあるコンテンツをパーツごとに切り分ける

<br>

5. さらに要素ごとに細かく切り分ける

<br>

6. テキストツールでタグやクラスを命名する  
   　`「tag .class #id」`

<br>

7. プロジェクトの共有  
   共有　 → 　リンクをコピー  
   公開情報を全員、閲覧に設定。
