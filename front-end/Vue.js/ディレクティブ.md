# ディレクティブとは
* Vue.jsで使用される特別な属性
* HTML要素の属性として使用し、`v-○○`の形式で記述される
* その要素の表示や挙動を制御する

<br>

# v-for
リスト(配列)のデータをループ表示できるディレクティブ
```
<script setup>
const fruitsList = [
  { id: 0, name: "リンゴ" },
  { id: 1, name: "苺" },
  {id: 2, name: "バナナ"},
];
</script>

<template>
  <ul>
    <li v-for="fruit in fruitsList" :key="fruit.id">{{ fruit.name }}</li>
  </ul>
</template>
```
* `v-for='変数名 in 並列名'`
* key属性と共に使うことが推奨される(なくても動く)
    * key属性によりリストの要素を一意に識別できる
    * `:key="配列名.カラム名"`でキー指定(要素が重複しない場合はそのまま要素を指定することも出来る)
* `{{ 配列名.カラム名 }}`でブラウザに表示

<br><br>

# v-if,v-else,v-show
## v-if,v-else
条件を指定して要素の表示・非表示を切り替えられる
```
<script setup>
const number = 5;
</script>

<template>
  <ul>
    <p v-if="number < 10">数字は10より小さい</p>
    <p v-else>数字は10より大きい</p>
  </ul>
</template>
```
* `v-if='条件'`で、条件がtrueの場合に表示される
* v-ifのある直後の要素でv-elseを使用すると、v-ifに当てはまらない場合にv-elseの要素が表示される

## v-show
v-ifと同じだが、v-elseに相当するものが無い
```
<template>
  <ul>
    <p v-show="number < 10">数字は10より小さい</p>
  </ul>
</template>
```

## v-ifとv-showの違い
* v-ifがfalseで非表示になる場合、要素自体が存在しない
* v-showが非表示になる場合、要素自体は存在し、`style="display: none"`で非表示になる
<br>

v-ifは非表示になる際、要素が削除されるため、再描画にかかるコストがv-showより大きくなる。  
    ⇒ 条件が頻繁に切り替わる場合はv-show、そうでない場合はv-ifを使うことで効率的に再描画できる。
