# セットアップ

1. リポジトリの設定
2. Docker の設定
3. Laravel のパッケージのインストール
4. .env ファイルの作成

## リポジトリの設定

### クローン

アプリケーションを作成するディレクトリまで移動し、GitHub からクローン。  
`git clone クローン元のURL`

※必要に応じてディレクトリ名変更  
`mv 元のディレクトリ名 変更後のディレクトリ名`  

必要に応じて.gitignoreの確認と修正。  
<br>

### リモートリポジトリの変更

自分の GitHub アカウントで新規リモートリポジトリを作成  
※公開・非公開設定は、基本 public（公開）。  
<br>

作成したリポジトリの URL をコピー  
クローンしたディレクトリまで移動し、URL 変更。

```
cd クローンしたリポジトリ
git remote set-url origin 作成したリポジトリのURL
```

下記コマンドで URL が変更できているか確認  
`git remote -v`  
<br>

現在のローカルリポジトリのデータをリモートリポジトリに反映

```
git add .
git commit -m "リモートリポジトリの変更"
git push origin main
```

GitHub にアクセスし、作成したリポジトリに変更が反映されているか確認。  
<br>

## Docker の設定

```
docker-compose up -d --build
code .
```

「Docker Desktop」からコンテナが作成出来ているか確認  
<br>

## Laravel のパッケージのインストール

```
docker-compose exec php bash
composer install
composer -v
```

バージョンが返ってきたら、インストール成功。  
<br>

## .env ファイル作成

クローンした.env.example を元に.env を作成。  
`cp .env.example .env`

<br><br>

# データベース接続

VS Code から.env ファイルを編集

```
// 前略

DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=laravel_user
DB_PASSWORD=laravel_pass

// 後略
```

ホスト名を mysql に変更  
docker-compose.yml で作成したデータベース名、ユーザ名、パスワードを記述。  
<br>

接続したデータベースの確認  
phpMyAdmin  
http://localhost:8080/
