# Laravelで“実は使わないほうがいい”命名まとめ

| 種類                  | NG例                                                               | 理由・どこで使われているか                           |
| ------------------- | ----------------------------------------------------------------- | --------------------------------------- |
| **Blade変数**         | `$message`                                                        | バリデーションや例外時に内部利用（@errorなど）              |
| 〃                   | `$errors`                                                         | `Illuminate\View\ViewErrorBag` が自動注入される |
| 〃                   | `$slot`, `$attributes`                                            | コンポーネントスロットで自動注入される                     |
| 〃                   | `$loop`                                                           | `@foreach` 構文で自動的に定義される（ループ番号など）        |
| 〃                   | `$component`, `$view`                                             | `@component`, `@include` 系内部で利用         |
| **モデル名／テーブル名**      | `User`, `Message`, `File`, `Session`, `Request`, `Cache`, `Event` | LaravelファサードやEloquent内部クラスと衝突しやすい       |
| **Livewire変数／メソッド** | `state`, `message`, `error`, `data`                               | Livewireが内部状態で予約済みのことがある                |
| **ルート名／コントローラ名**    | `HomeController`, `Controller@index`, `LoginController`           | FortifyやJetstream導入時に被る可能性あり            |
| **Bladeファイル名**      | `errors.blade.php`, `message.blade.php`                           | システム標準エラー表示と名前が衝突                       |
<br><br>

## 命名のコツ
1. アプリ固有の接頭辞をつける...例：$chatMessage, $userProfile, $taskItem
2. 複数形を避ける（特に$errors, $messages）
3. Livewireでは $state, $data, $message は避ける
<br><br>

# Laravel内部の“ややこしい変数TOP3”
1. $message（例外・バリデーション・ログ全部で使われる）
2. $errors（エラーバッグ）
3. $loop（foreach内部で自動生成）
